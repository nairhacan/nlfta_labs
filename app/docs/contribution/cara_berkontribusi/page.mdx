## Alur Kontribusi Langkah-demi-Langkah

### **Step 1: Temukan Issue yang Sesuai**

1. Kunjungi **[Issues page NLFTs](https://github.com/NLFTs/NLFTs/issues)**
2. Filter dengan label berikut untuk memulai:

   ```bash
   # Label yang direkomendasikan untuk pemula:
   - `good first issue`     # Cocok untuk kontributor baru
   - `help wanted`          # Membutuhkan bantuan komunitas
   - `documentation`        # Perbaikan dokumentasi
   - `bug`                  # Bug yang perlu diperbaiki
   ```

3. **Baca deskripsi issue dengan seksama**
4. **Komentari** issue tersebut untuk mengklaimnya (misal: "I'd like to work on this!")

### **Step 2: Fork Repository (WAJIB untuk Non-Anggota)**

Karena Anda bukan anggota organisasi **NLFTs**, Anda harus membuat fork:

1. **Buka [Repository NLFTs](https://github.com/NLFTs/NLFTs)**
2. **Klik tombol "Fork"** di kanan atas:
   
   ![Fork Button](https://docs.github.com/assets/cb-23088/mw-1440/images/help/repository/fork-button.webp)
   
3. **Pilih akun Anda** sebagai destination
4. **Tunggu** hingga proses selesai (beberapa detik)

**Hasil:** Anda sekarang memiliki salinan pribadi di:
```
https://github.com/username-anda/NLFTs
```

### **Step 3: Clone Repository ke Lokal**

```bash
# Clone fork Anda (GANTI 'username-anda' dengan username GitHub Anda)
git clone https://github.com/username-anda/NLFTs.git

# Masuk ke direktori proyek
cd NLFTs

# Tambahkan upstream repository (repository utama NLFTs)
git remote add upstream https://github.com/NLFTs/NLFTs.git

# Verifikasi remotes
git remote -v
# Harus menampilkan:
# origin    https://github.com/username-anda/NLFTs.git (fetch)
# origin    https://github.com/username-anda/NLFTs.git (push)
# upstream  https://github.com/NLFTs/NLFTs.git (fetch)
# upstream  https://github.com/NLFTs/NLFTs.git (push)
```

### **Step 4: Setup Environment Development**

```bash
# Install dependencies (kami menggunakan pnpm)
pnpm install

# Jalankan development server
pnpm dev

# Buka browser dan kunjungi http://localhost:3000
# Verifikasi bahwa proyek berjalan dengan benar

# (Opsional) Jalankan test untuk memastikan setup berhasil
pnpm test
```

### **Step 5: Buat Branch untuk Perubahan Anda**

**JANGAN** bekerja di branch `main` langsung! Selalu buat branch baru:

```bash
# Format: type/deskripsi-singkat
# Contoh:
git checkout -b feat/add-dark-mode
git checkout -b fix/login-error-422
git checkout -b docs/update-readme-installation

# Konvensi type yang kami gunakan:
# - feat:     Fitur baru
# - fix:      Perbaikan bug
# - docs:     Perubahan dokumentasi
# - style:    Formatting, semi-colon, dll (tidak mengubah logic)
# - refactor: Perbaikan kode tanpa mengubah perilaku
# - test:     Menambah atau memperbaiki test
# - chore:    Perubahan build process, tooling, dll
```

##  Standar Kode

### **JavaScript/TypeScript**
- Gunakan **TypeScript** untuk file baru
- Ikuti [Airbnb JavaScript Style Guide](https://github.com/airbnb/javascript)
- Gunakan **ES6+ features** ketika memungkinkan
- **Naming conventions:**
  - `camelCase` untuk variables/functions
  - `PascalCase` untuk components/classes
  - `UPPER_SNAKE_CASE` untuk constants

### **Nuxt.js/Next.js**
- Gunakan **functional components** dengan hooks
- Implement **proper TypeScript interfaces/props**
- Gunakan **modular CSS/SCSS** atau CSS-in-JS
- Optimize untuk **performance** (React.memo, useMemo, useCallback)

### **Commit & PR Best Practices**
1. **Satu PR = satu fitur/fix**
2. **Jaga PR size manageable** (< 500 lines ideal)
3. **Update documentation** bersamaan dengan code changes
4. **Include tests** untuk perubahan signifikan
